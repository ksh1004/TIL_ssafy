# CLI

1. GUI : Graphic User Interface, 그래픽을 통해 사용자와 컴퓨터가 상호작용
2. CLI : Command Line Interface, 명령어를 통해 사용자와 컴퓨터가 상호작용

  가. CLI 장점
    1) 효율성 : 모든 작업을 빠르게 수행 가능
    2) 정밀한 제어
    3) 자동화 : 명령어 한 줄로 단순, 반복 작업 자동화 가능
    4) 표준 환경

  나. 문법
    1) . : 현재 디렉토리(작업 위치)
      * 상대경로
    2) .. : 상위 디렉토리
      * 상대경로
    3) touch : 파일 생성
    4) mkdir : 새 디렉토리 생성
      * **m**a**k**e **dir**ectory의 줄임말
    5) ls : 현재 폴더의 파일을 조회
      * list의 줄임말
    6) mv : 파일을 이동
      * move의 줄임말
      ** mv git.md day01/ (명령어 옮길파일 옮길폴더)
    7) cd : 디렉토리 이동
      * **c**hange **d**irectory의 줄임말
    8) code : 파일 실행
      * tab키를 누르면 자동완성 가능
